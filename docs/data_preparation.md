# Data Preparation

This document walks through preparing your 6,000 annotated fungal genomes for use with FungalClassifier. It covers the expected directory structure, required annotation types, metadata format, and suggested QC steps.

---

## Directory Structure

```
data/
├── raw/
│   ├── genomes/           ← One FASTA per genome: {genome_id}.fna
│   ├── annotations/
│   │   ├── pfam/          ← hmmscan domtblout: {genome_id}.domtblout
│   │   ├── dbcan/         ← dbCAN overview: {genome_id}_overview.txt
│   │   ├── antismash/     ← antiSMASH JSON: {genome_id}.json
│   │   ├── repeatmasker/  ← RepeatMasker out: {genome_id}.out
│   │   ├── gff/           ← Gene annotations: {genome_id}.gff3
│   │   ├── kegg/          ← KO annotations: {genome_id}_ko.tsv
│   │   └── fimo_work/     ← (auto-generated by motif pipeline)
│   ├── phylogeny.nwk      ← Newick tree, tip labels = genome_id
│   └── metadata.tsv       ← See format below
└── features/              ← (auto-generated by 01_build_features.py)
```

**Important:** Genome IDs must be consistent across all files. The stem of each FASTA filename (without extension) is used as the genome ID everywhere.

---

## Metadata Format

`data/raw/metadata.tsv` must be tab-separated with `genome_id` as the first column:

```
genome_id	taxonomy_kingdom	taxonomy_phylum	taxonomy_class	taxonomy_order	taxonomy_family	taxonomy_genus	taxonomy_species	ecological_niche	lifestyle	tree_label
GCA_000001	Fungi	Ascomycota	Sordariomycetes	Hypocreales	Nectriaceae	Fusarium	oxysporum	plant_pathogen	necrotroph	GCA_000001
GCA_000002	Fungi	Basidiomycota	Agaricomycetes	Agaricales	Amanitaceae	Amanita	muscaria	ectomycorrhizal	biotroph	GCA_000002
```

**Required columns:** `genome_id`, at least one taxonomy column, at least one ecological target column.

**Ecological niche categories** (standardise to these): `saprotrophic`, `plant_pathogen`, `ectomycorrhizal`, `arbuscular_mycorrhizal`, `endophytic`, `animal_pathogen`, `lichenicolous`, `mycoparasitic`, `unknown`.

**Lifestyle categories:** `saprotroph`, `obligate_biotroph`, `hemibiotroph`, `necrotroph`, `unknown`.

## Annotation Pipeline

### 1. Protein domain annotation (Pfam)

```bash
# Index genome
makeblastdb -in genome.fna -dbtype nucl

# Predict proteins (if not already done)
augustus --species=fusarium genome.fna > genome.gff3
getAnnoFasta.pl genome.gff3  # produces genome.aa

# Run hmmscan against Pfam-A
hmmscan \
    --domtblout annotations/pfam/genome_id.domtblout \
    --cpu 8 \
    -E 1e-5 \
    /path/to/Pfam-A.hmm \
    genome.aa
```

Download Pfam-A: `wget https://ftp.ebi.ac.uk/pub/databases/Pfam/releases/Pfam37.0/Pfam-A.hmm.gz`

### 2. CAZyme annotation (dbCAN)

```bash
run_dbcan \
    genome.aa \
    protein \
    --out_dir annotations/dbcan/genome_id/ \
    --db_dir /path/to/dbcan_db \
    --tools hmmer diamond hotpep \
    --cpu 8
cp annotations/dbcan/genome_id/overview.txt annotations/dbcan/genome_id_overview.txt
```

Download dbCAN database: `dbcan_build --cpus 8 --db-dir /path/to/dbcan_db`

### 3. BGC annotation (antiSMASH)

```bash
antismash \
    genome.fna \
    --output-dir annotations/antismash/genome_id/ \
    --genefinding-tool prodigal \
    --cb-general \
    --cb-knownclusters \
    --cpus 8

# antiSMASH outputs a JSON: annotations/antismash/genome_id/genome_id.json
cp annotations/antismash/genome_id/genome_id.json annotations/antismash/genome_id.json
```

### 4. Repeat annotation (RepeatMasker)

```bash
RepeatMasker \
    -pa 8 \
    -species fungi \
    -dir annotations/repeatmasker/ \
    -gff \
    genome.fna
# Produces: genome.fna.out → rename to genome_id.out
```

### 5. Promoter motif scanning (FIMO / JASPAR)

The motif pipeline is automated via `01_build_features.py` when `--blocks motifs` is specified. It requires:
- A GFF3 annotation file per genome in `annotations/gff/{genome_id}.gff3`
- The JASPAR 2024 fungi MEME PWM file

```bash
# Download JASPAR 2024 fungi PWMs
wget https://jaspar.elixir.lu/download/data/2024/CORE/JASPAR2024_CORE_fungi_non-redundant_pfms_meme.txt \
    -O data/raw/annotations/jaspar/JASPAR2024_CORE_fungi_non-redundant_pfms_meme.txt

# Index each genome for bedtools
for fna in data/raw/genomes/*.fna; do
    samtools faidx $fna
done
```

### 6. KEGG ortholog annotation

```bash
# Using KofamScan (recommended for fungi)
exec_annotation \
    -f mapper \
    -o annotations/kegg/genome_id_ko.tsv \
    --cpu 8 \
    --profile /path/to/kofam_profiles \
    --tmp-dir tmp/genome_id \
    genome.aa
```

Download KofamScan profiles: https://www.genome.jp/tools/kofamkoala/

## Phylogenetic Tree

FungalClassifier expects a rooted or unrooted Newick tree with tip labels matching `genome_id` exactly.

**Recommended approach for 6,000 genomes:**
1. Extract single-copy orthogroups with OrthoFinder
2. Align with MAFFT, trim with trimAl
3. Infer tree with IQ-TREE2 (GTRGAMMA, 1000 ultrafast bootstraps)
4. Root with mad (minimal ancestor deviation) or using a known outgroup

```bash
# IQ-TREE2 (recommended for large datasets)
iqtree2 \
    -s alignment.phy \
    -m GTR+G \
    -B 1000 \
    -T AUTO \
    --prefix phylogeny

cp phylogeny.treefile data/raw/phylogeny.nwk
```

Alternative: use a pre-built fungal phylogeny from NCBI taxonomy or FungiDB and prune to your genome IDs using `prune_tree_to_genomes()`.

## Quality Control

Before running the pipeline, check:

### Genome completeness
```bash
# Run BUSCO on all genomes
busco -i genome.fna -l fungi_odb10 -o busco_output -m genome --cpu 8
# Keep genomes with >80% BUSCO completeness
```

### Annotation completeness
```python
from fungal_classifier.utils.io import load_metadata, discover_annotation_files

metadata = load_metadata("data/raw/metadata.tsv")
pfam_paths = discover_annotation_files("data/raw/annotations/pfam", ".domtblout")
missing = set(metadata.index) - set(pfam_paths.keys())
print(f"Missing Pfam annotations: {len(missing)} genomes")
print(missing)
```

### Metadata consistency
```python
import pandas as pd
metadata = pd.read_csv("data/raw/metadata.tsv", sep="\t")

# Check for required columns
required = ["genome_id", "taxonomy_order", "ecological_niche"]
missing_cols = [c for c in required if c not in metadata.columns]
print(f"Missing columns: {missing_cols}")

# Check tree label consistency
from fungal_classifier.evaluation.phylo_cv import load_tree, get_tree_tip_names
tree = load_tree("data/raw/phylogeny.nwk")
tree_tips = set(get_tree_tip_names(tree))
meta_ids = set(metadata["genome_id"])
print(f"In tree but not metadata: {len(tree_tips - meta_ids)}")
print(f"In metadata but not tree: {len(meta_ids - tree_tips)}")
```

## Minimum Viable Dataset

If you don't have all annotations, the pipeline degrades gracefully:

| Available annotations | Blocks usable |
|---|---|
| Genome FASTA only | `kmer` |
| + Pfam domtblout | `kmer`, `domains` |
| + dbCAN, antiSMASH | `kmer`, `domains`, `pathways` |
| + RepeatMasker | all except `motifs` |
| + GFF3 + JASPAR | all blocks |

For a first run, `kmer` + `domains` + `cazyme` is typically sufficient to achieve high accuracy on taxonomy classification, and those three are the fastest to generate.
